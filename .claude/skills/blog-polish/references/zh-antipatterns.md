# Chinese Anti-Patterns Reference (中文反模式)

中文版的核心问题不是 "AI 通用味"，而是 **机器翻译腔（machine-translation stiffness）**——直接从英文句式照搬到中文。这些规则专门针对这个问题。

## 1. 代词链 / Pronoun chains

**英文需要 "you, your, the, it"，中文不需要。**

中文会大量省略代词。"你向你大脑宣告的是" 是直译，原生中文会说 "你其实是在告诉自己"。

| 翻译腔 ❌ | 原生中文 ✅ |
|---|---|
| 你向你的大脑宣告的是 | 你其实是在告诉自己 / 你这是在跟自己说 |
| 它会自动跑反馈回路把你拉回这个目标上 | 它会自动跑反馈循环，把你拽回这个目标 |
| 当你说"我想要 X"的时候 | 你说"我想要 X"的时候 |
| 你的大脑会主动给你制造阻力 | 大脑会主动给你制造阻力 |

**Rule:** 扫描每一串 "你的 / 他的 / 它的 / 我的"。问：删掉这个代词，意思变了吗？没变就删。

## 2. 主从结构 / Long nested sentences

**英文偏好嵌套从句，中文偏好短句串联。**

如果一句话里出现 3 个以上 "的"，或一个从句套着另一个从句，重写成 2-3 个短句。

| 翻译腔 ❌ | 原生中文 ✅ |
|---|---|
| 这是一个挺复杂的、与你的过去经历有关的、需要花时间消化的问题。 | 这问题挺复杂。跟你的过去有关。得花时间消化。|
| 当我意识到我之前一直在用错误的方式思考这件事的时候，我才发现我浪费了很多时间。 | 我发现我之前思考的方式根本就是错的。回头一看，浪费了不少时间。|

**Rule:** 一句话里 3+ 个 "的" → 重写。3+ 个分句嵌套 → 拆成两句。

## 3. 缺少语气词 / Missing modal particles

中文博客 / 公众号 tone **依赖语气词**。一段没有 吧/啊/嘛/呢/了/吗 = 机器翻译。

常用语气词及功能：
- **了** — 完成、变化（"它就漂回去了"）
- **吧** — 缓和、建议（"不缺钱的人今天会怎么过吧"）
- **啊** — 感叹、亲近（"挺玄的啊"）
- **嘛** — 解释、不耐烦（"我就 i 人嘛"）
- **呢** — 疑问、对比（"你呢？"）
- **吗** — 是非问

**Rule:** 每段至少 1 个语气词。如果整段没有 → 找 1-2 个自然的位置加上。不要硬塞。

## 4. 破折号过度 / Em-dash overuse

**中文用破折号（——）远比英文少。** 英文 em-dash 自然，中文 用得多了像翻译。

替代方案：
- **冒号(：)** — 引出解释 / 例子
- **换行 + 段落** — 强调
- **中文括号（）** — 补充说明
- **句号** — 直接断句

| 翻译腔 ❌ | 原生中文 ✅ |
|---|---|
| 这件事——你想想就明白——本质上很简单 | 这件事你想想就明白：本质上很简单 |
| Maltz 借了一个词——servo——来形容 | Maltz 借了一个词，叫 servo |
| 不是因为读书快乐——是因为这就是他 | 不是因为读书快乐。是因为这就是他。|

**Rule:** 每段不超过 1 个 ——。一段里超过 2 个 → 替换。

## 5. 直译连接词 / Translated connectors

**有些英文连接词在中文里有更自然的对应。**

| 翻译腔 ❌ | 原生中文 ✅ |
|---|---|
| 事实上 / 实际上 | 其实 / 讲白了 / 说白了 |
| 此外 / 另外 | 还有就是 / 而且 |
| 因此 | 所以 |
| 然而 | 可是 / 但 / 可 |
| 换句话说 | 也就是说 / 这么说吧 |
| 与此同时 | 同时 / 顺带 |
| 不仅...而且 | 又...又 / 不光...还 |
| 不是 X，而是 Y | 不是 X，是 Y / 说白了不是 X，是 Y |

**Rule:** "事实上" / "因此" 出现 → 大概率是直译，换更口语的。

## 6. 直译 "It's not X. It's Y." 

英文这个对仗句式很常见，中文直译 "这不是 X，这是 Y" 会反复出现，读起来非常机械。

变体：
- "X？不是。是 Y。"
- "说白了不是 X，是 Y。"
- "它不像 X。它更像 Y。"
- 拆成两句独立的话
- 用 "其实是 Y / 真正是 Y" 转向

**Rule:** 整篇文章不超过 2 次 "这不是 X，是 Y"。

## 7. Title 翻译腔

中文标题 **不要用 4 字文言成语**（"曳尾涂中"、"知行合一"、"破而后立"）除非文章本身就是讲那个成语。

✅ 现代白话：
- "做一只泥里的乌龟"
- "你不是在找自己，是在写自己"
- "为什么聪明人都在 FIRE"

❌ 太文言：
- "曳尾涂中"
- "无问西东"
- "知止而后有定"

**Rule:** 中文标题 ≤ 12 字，全现代汉语，不用文言成语。

## 8. 保留英文术语

Zach 在中英文夹杂时有自然规律——某些技术 / 专业词留英文比翻译更自然。

**保留英文：**
- 技术术语：`servo`, `prompt`, `LLM`, `harness`, `agent`, `persona`, `backlog`, `Slack`, `PR`, `commit`
- 网络 / 流行词：`flag`（立 flag）, `i 人`/`e 人`, `BGM`, `IP`, `OKR`, `KPI`
- 专有名词：人名（Naval, Maltz, Buffett）、书名（《Atomic Habits》、《Mindset》）、公司名（McKinsey、LinkedIn）

**翻译成中文：**
- 抽象概念：identity → 身份 / 自我形象（不用 "认同"）
- 通用动词：execute → 执行 / 上场（看语境）
- 情绪 / 状态：frustrated → 烦 / 受不了

**Rule:** Zach 之前的文章里某个词怎么处理过 → 保持一致。

## 9. 段落节奏

中文段落 **比英文段落更短**。

- 一句话一段是常见的（强调、节奏）
- 两段之间空行，留呼吸
- 长段（>5 句）很少出现，除非在做密集论证

**Rule:** 任何一段超过 5 句 → 看能不能拆。

## 10. Self-check（中文部分）

报告 polish 完成前，对中文部分核对：

- [ ] 代词链：没有 "你的 / 他的 / 它的" 串
- [ ] 嵌套句：没有 "3+ 的" 或 3+ 从句嵌套
- [ ] 语气词：每段至少 1 个
- [ ] 破折号：每段 ≤ 1 个 ——
- [ ] 连接词：没有 "事实上 / 因此 / 然而" 等翻译腔
- [ ] "不是 X，是 Y" 对仗：整篇 ≤ 2 次
- [ ] 标题：现代汉语，≤ 12 字，不用文言
- [ ] 英文术语：保留 Zach 一贯保留的，翻译该翻译的
- [ ] 段落：超过 5 句的段落已拆
- [ ] 双语对称：和 EN 版表达同一个论点，但**不是逐句翻译**
